# CRIS-LOOP 完整闭环开发系统

## 核心理念

**每次迭代走完所有阶段，循环往复，持续改进。**

```
┌─────────────────────────────────────────────────────────────┐
│                   CRIS-LOOP 完整闭环                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│    ┌─────────────────────────────────────────────────┐      │
│    │                                                 │      │
│    │   P ──→ Pr ──→ A ──→ C ──→ B ──→ Q ──→ (循环) │      │
│    │   │      │      │      │      │      │         │      │
│    │   ▼      ▼      ▼      ▼      ▼      ▼         │      │
│    │  规划   优先级   API   组件   编码   质量        │      │
│    │  Plan  Priority API  Comp   Build  Quality       │      │
│    │                                                 │      │
│    └─────────────────────────────────────────────────┘      │
│                         │                                   │
│                         ▼                                   │
│                   GitHub 同步推送                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

## 6阶段详解

### P - Plan（规划）
- 功能需求描述
- 目标与范围定义
- 资源评估

### Pr - Priority（优先级）
- 优先级排序
- 依赖关系分析
- 风险评估

### A - API（API设计）
- 接口设计
- 数据模型设计
- 请求/响应格式

### C - Component（组件设计）
- 前端组件结构
- 状态管理方案
- 页面路由设计

### B - Build（编码构建）
- 代码实现
- 单元测试
- 集成测试

### Q - Quality（质量审查）
- 代码质量检查
- 构建验证
- 文档更新

---

## 执行规则

### 每次循环
```
1. 执行 P-Pr-A-C-B-Q 全部6个阶段
2. 完成后进入下一轮循环
3. 每次循环都产出可运行的代码
```

### GitHub 同步
| 时机 | 操作 |
|------|------|
| 每次循环完成 | 更新 PROGRESS.md |
| 新功能完成 | 更新 CHANGELOG.md |
| 架构变更 | 更新 ARCHITECTURE.md |
| 所有变更 | 实时推送到 GitHub |
