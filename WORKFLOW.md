# Vox (内容生成Agent) - 自主工作流

## 自主循环机制

> 核心理念：像 Sigma 学习，自主循环、自主进化、持续改进

---

## 一、每次接收到任务的循环流程

```
┌─────────────────────────────────────────────────────────────┐
│  接收任务                                                    │
│  输入：产品信息 + 目标平台                                   │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤0: 检查功能框架                                         │
│  - 读 ../_FEATURE-FRAMEWORK.md                              │
│  - ⚠️ 只读 "Vox 任务清单" 部分，不要读其他项目的功能       │
│  - ⚠️ 只添加 Vox 自己的功能，不要借鉴其他项目                │
│  - 定期自主循环时检查是否有新的GitHub发现需要添加           │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤1: 理解任务                                             │
│  - 读 ARCHITECTURE.md 理解当前架构                        │
│  - 读 INSTRUCTIONS.md 理解产品定位                         │
│  - 问自己：这个产品适合什么内容风格？                        │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤2: 规划实现                                            │
│  - 需要改哪些模块？                                          │
│  - 需要新增哪些函数？                                        │
│  - 需要参考 Sigma 哪些 Skills/Agents？                      │
│  - 更新 ARCHITECTURE.md（新增模块）                         │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤3: GitHub 学习（如果需要）                            │
│  - 去 _reference/ 研究参考项目                             │
│  - 搜索 GitHub 相关项目找灵感                               │
│  - 如果有新的好的参考，clone 到 _reference/                │
│  - 记录学到的东西到 _LEARNING.md                           │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤4: 实现代码                                            │
│  - 按 ARCHITECTURE.md 的模块实现                           │
│  - 遵循 Sigma 的 Skills 规范                              │
│  - 遵循 ARCHITECTURE.md 的格式写注释                       │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤5: 测试验证                                            │
│  - 运行测试用例                                             │
│  - 文案质量如何？违规词检测通过？                            │
│  - 配图建议是否合适？                                        │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤6: 更新文档                                            │
│  - 更新 ARCHITECTURE.md（如果新增模块）                    │
│  - 更新 _LEARNING.md（记录学到的）                         │
│  - 更新 PROGRESS.md（记录本轮做了什么）                    │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤7: 自我审计                                            │
│  - 代码质量：遵循规范？模块化？                              │
│  - 功能完整：所有功能都实现了？                              │
│  - 文档完整：注释、ARCHITECTURE、SKILL都更新了？            │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤8: 版本管理                                            │
│  - git add . && git commit -m "描述本次完成的工作"        │
│  - git push 推送到 GitHub 仓库                            │
│  - 确保所有重要更改都已提交和推送                          │
└─────────────────────────────────────────────────────────────┘
                        ↓
                        ↓ 循环结束
```

## 二、定期自主循环（无新任务时）

当没有新任务时，每轮自动执行：

```
┌─────────────────────────────────────────────────────────────┐
│  定期循环 (每轮自动执行)                                    │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤0: 再检查功能框架                                      │
│  - ⚠️ 只确认 "Vox 任务清单" 是否有更新                      │
│  - ⚠️ 不要看其他项目的功能清单                              │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  1. GitHub 趋势扫描                                        │
│  - 搜索 AI content generation / marketing copy 相关项目    │
│  - 找有没有更好的内容生成相关工具                            │
│  - 有就 clone 到 _reference/                              │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  2. Sigma Skills 学习                                      │
│  - 读 pm-agent-forge/skills/ 相关 Skills                   │
│  - 看有没有可以应用到内容生成的                             │
│  - 有就更新自己的实现                                       │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 代码优化                                                │
│  - 检查有没有重复代码                                       │
│  - 检查有没有可以抽象的模块                                  │
│  - 检查有没有更优雅的实现方式                                │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 功能增强                                                │
│  - 看参考项目有没有新功能                                    │
│  - 实现一个小的功能增强                                      │
│  - 记录到 PROGRESS.md                                      │
└─────────────────────────────────────────────────────────────┘
```

## 四、动态跟踪各种更新

**重要**: 不能写死，要动态跟踪 Sigma、其他项目、GitHub 的变化。

### 4.1 Sigma Skills/Agents 动态发现
每轮定期循环时扫描：
```
D:/PM-AI-Workstation/01-ai-agents/pm-agent-forge/skills/   # Skills
D:/PM-AI-Workstation/01-ai-agents/pm-agent-forge/agents/   # Agents
```
- 发现新的相关 Skills/Agents 时，更新到本文档
- 发现 Skills 有更新时，以官方为准
- 本项目相关领域：`marketing/`、`content/`、`common/`

## 五、文档更新规范

### ARCHITECTURE.md 更新
新增模块时：
```markdown
## module-name 模块名

### 类型
- `TypeName` - 描述

### 函数
- `functionName(args) -> ReturnType` 描述

### 测试
- should do something
```

### _LEARNING.md 更新
学到新东西时：
```markdown
## 第N轮学习

### GitHub 新发现
- [项目名] - 简介 - 可用于：xxx

### Sigma Skills 新应用
- [Skill名] - 应用场景：xxx

### 代码优化
- 优化了 xxx 模块 - 原因：xxx
```

### PROGRESS.md 更新
每轮结束时：
```markdown
## 第N轮

### 完成
- [功能1]
- [功能2]

### 学习
- [学到的1]
- [学到的2]

### 问题
- [遇到的问题]

### 下轮计划
- [计划1]
```

## 六、循环终止条件

可以终止的情况：
1. 用户说"满意了"或"可以了"
2. 循环 10 次后自动暂停，等待用户新任务
3. 遇到无法解决的技术难题，记录并暂停

---

## 七、与其他项目的协调

如果发现某个功能可以在多个项目复用：
1. 考虑是否值得抽取到 Sigma
2. 如果是，更新 pm-agent-forge/skills/
3. 各项目从 Sigma 获取 Skills
